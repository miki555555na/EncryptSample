格子　
　ベクトル空間R^nにおいて、線型独立なm個のベクトルb1,...,bmの線形係数による線型結合で表される集合
　L = { Σ aibi | ai ∈ Z }

イメージ格子・カーネル格子
　イメージ格子：行列Aの列ベクトルの線型結合(にqの倍数を加えたもの)で生成される格子
　　　Λq(A) = {y ∈ Z^n | y ≡ As mod q (for some s) }
　カーネル格子：行列Aを掛けると(法qで)ゼロベクトルになるようなベクトルの集合。
　　　Λq^⊥(A) = {x ∈ Z^m | Ax ≡ 0 mod q }

LWE問題　提案した最初の論文：「On Lattices, Learning with Errors, Random Linear Codes, and Cryptography」
　b = As + e (mod q)
　未知のベクトルs ∈ Zq^n があるとき、ランダムな行列Aと誤差ベクトルeを用いて計算する。
　計算されたbとAをペアで公開鍵(A,b)として与えられたとき、秘密鍵sを解く問題。
　誤差eを加えることで、sを解くことを難しくしている。

　<幾何学的なイメージ>
　1. 格子の点 As (mod q)は、イメージ格子Λq(A)上の点を指している。
　2. そこに小さなノイズeを加えることで、「格子点から少しだけズレた場所」に移動する。
　3. 「ズレた地点 b」から見て、「元々どの格子点にいたか？」を当てるのがLWE問題

　安全性：ノイズがある中で連立一次方程式の解法の難しさ


トラップドア　提案した最初の論文：「How to Use a Short Basis: Trapdoors for Hard Lattices and New Cryptographic Constructions」
　ある格子に対する「短い基底」
　
　公開情報(平均ケース)：外から見ると、一様ランダムな行列Aに見える格子
　秘密情報(トラップドア)：その格子に対する「短い基底」トラップドアを知っていると、その格子におけるSVPやCVPを解くことができる。

　トラップドア付きの格子をLWE問題と組み合わせる
　・攻撃者にとっては、公開されているAは難しいLWEインスタンス(平均ケース)にしか見えない。
　・「短い基底」を持つ人にとっては、この格子の上で「離散ガウスサンプリング」という手法を用いることで、特定の分布に従う短いベクトルを生成することができる。

　~IDベース暗号への応用~
　1.PKGはマスター公開鍵Aとマスター秘密鍵Tを生成する。
　　・マスター公開鍵 A ∈ Zq^{nxm}：この行列は一様分布と区別がつかないが、特定の「短い基底」をトラップドアとして持つ。
　　・マスター秘密鍵 T ：「短い基底」である行列T 
　2.IDから公開鍵を作る
　　・ユーザーのIDをハッシュ化して、格子空間上のベクトル(点)uに変換する。
　3.秘密鍵の発行
　　・マスター秘密鍵Tを使って、次を満たす短いベクトル e ∈ Z^m をサンプリングする。(Tがバレないようにeを作るためにGPVの離散ガウスサンプリング使用)
　　　　Ae ≡ u (mod q)
　　・eがユーザーの秘密鍵になる。平均ケースにおいて、Aとuからeを知ることはできない。
　4.暗号化・復号
　　・Ae ≡ u (mod q)が解けないことを利用して、暗号化・復号を行う。


　　<損失性トラップドア関数>　提案した最初の論文：「Lossy Trapdoor Functions and Their Applications」
　　　暗号化されたものが正規の入力でない時、ダミー関数として損失性トラップドア関数を使用することで、CCA安全性を実現。
　　　単射モードでは入力と出力が1対1に対応するため、トラップドアがあれば出力から入力を効率的に復元できる(Ae = uで入力A,uからeを効率的に出力)が、
　　　損失モードでは入力に関する情報を「失わせる」ため、出力の候補が入力よちも大幅に小さくなる。その結果入力を特定できなくなる。

　　 ~IDベース暗号への応用~
　　　・入力が正規のものであるかを判別する場合、「タグt」を使用するが、IBEにおいてメッセージ受信者のIDそのものがタグtになる。
　　　・「正規のID（タグ）で暗号化された時だけ、手持ちの秘密鍵で復号できる」ような数理的トラップを、公開パラメータである行列Aにあらかじめ仕込んでおく
　　　　　公開パラメータ：行列A(ランダム)、行列G(特殊な構造を含む)
　　　　　タグの注入：暗号化の際に送信者が新しい行列At = A - t・G
　　　　　暗号化：u = s・At + e (sは乱数ベクトル、eは小さな誤差) ← これにメッセージを載せて暗号文とする
　　　・単射モードでは特定のタグt*(ターゲットとなるID)においてのみ、トラップドアが機能するようにAを設定する。A = t*・G + (トラップドアを持つ行列) のように生成、At* = A - t*・Gを計算した結果、トラップドアを持つ行列が残るため、秘密鍵(トラップドア)を持つ受信者は正しく復号できる。
　　　・損失モードではAを A = t*・G +（純粋なLWEサンプル、つまりノイズ）として生成する。送信者がt*で暗号化すると、At* = A - t*・Gの結果は、ただのランダムなノイズの塊になる。


　　<トラップドアの効率的な利用・活用方法の提案1> 提案した論文：「Trapdoors for Lattices: Simpler, Tighter, Faster, Smaller」
　　　ガジェット行列G：トラップドアを持たなくても、逆計算が簡単という性質を持つ行列。
　　　　　　　　　　　　一般的には、qを法とする数aを2進展開(またはk進展開)するようなベクトルを並べた形。
　　　　　　　　　　　　G・x = uとなる短いベクトルxを探す際に、基底の計算は必要なく、単に値を分解するだけで解(短いベクトル)が求まる。

    　　PKG（鍵生成局）は、この「扱いやすい G」を「ランダムに見える行列」の中に隠す。
        1. ランダムな行列 ¯A を生成する（これはトラップドアを持たない）。
        2. 小さな要素（0, 1, -1など）からなるランダムな行列Rを生成する。このRが「トラップドア（秘密鍵）」 になる。
        3. 公開鍵Aを以下の形式で連結して作成する。A = [¯A | G - ¯AR]  (mod q)
        このAは、Rが小さければ LWE問題の仮定により、完全なランダム行列と区別がつかない。

        トラップドア（R）を用いた逆計算の仕組み
        もし、あるベクトル v に対して A・v = u となる短いvを探したい場合、トラップドアRを持っていれば、計算を「簡単な G」の空間に引きずり込むことができる。
        1. A = [¯A|B] （ただし B = G - ¯AR）と置く。
        2. ベクトル v を二つのパーツ [v1, v2]^T に分けると、A・v = ¯Av1 + (G - ¯AR)v2 = u となる。
        3. ここで、¯A(v1 - Rv2) + Gv2 = u と変形
        4. もし v1 = Rv2 となるように選べば、¯A の項が完全に消え、Gv2 = u という「簡単な問題」に帰着する。
        5. ガジェット行列 G の性質を使って短い v2 を求めれば、自動的に v1 も短くなり（R が小さいため）、結果として A 全体に対する短い解 v が手に入る。


　　<GPV方式> 提案した論文：「How to Use a Short Basis: Trapdoors for Hard Lattices and New Cryptographic Constructions」
　　　ハッシュして署名する(Hash-and-Sign)流れを格子暗号で実現したもの。
　　　署名の手順　
　　　　1.メッセージをハッシュ関数によって格子の空間上の点にマッピングする。
　　　　2.秘密鍵(トラップドア)を使って、そのターゲットに「非常に近い格子の点」を見つける。
　　　　3.その点との差分を署名とする。
　　　署名の検証
　　　　　署名が十分に短いベクトルであること、公開鍵を用いて計算した結果がメッセージのハッシュ値と一致することを確認。
　　　
　　　GPV方式：署名から秘密鍵の情報が漏洩してしまう脆弱性（アルゴリズムの出力に偏りが出る問題）を離散ガウス分布を用いることで解決(ターゲットに「一番近い点」を常に選ぶと秘密鍵の構造が漏れてしまう)。
　　　　　　　　「署名から秘密鍵に関する情報が一切漏れない」ことを数学的に証明した。

　　　~IDベース暗号への応用~
　　　　公開鍵（ID）を自由に設定できるのに、それに対応する秘密鍵を安全に発行できる仕組みを作った
　　　　　1. マスター公開鍵A」と「マスター秘密鍵T」の生成時
　　　　　2. IDをハッシュ化して生成されたベクトルuを用いて、Ae = u(mod q)を満たすeを見つける際に使用。「マスター秘密鍵Tの情報を漏らさない統計的な分布」を持つ短いベクトルeを選び出す。

Ring-LWE問題 提案した論文：「On Ideal Lattices and Learning with Errors Over Rings」
　LWEでベクトルと行列を用いていた計算を、多項式環上の演算に置き換えたもの。
　公開されたRq(例：n次多項式をx^n+1と素数qで割ったあまりの集合)において、秘密の多項式sと誤差多項式eを用いたペア(a,b = a・s + e)を、完全にランダムなペアと区別できるか、あるいはbからsを特定できるかという問題。
　多項式同士の掛け算「高速フーリエ変換(FFT)」などを用いることで高速に計算でき、公開鍵のサイズも従来のLWEの約1/nに削減できる。

　　Rq = Zq[x]/<x^n+1> 　（nは2のべき乗、qは素数）

　<安全性の証明>　2段階
　もしRing-LWEを解く効率的なアルゴリズムが存在するならば、「イデアル格子」における最短ベクトル問題(SVP)などの困難な問題を解く量子アルゴリズムが存在するこをを示した。
　・探索問題の困難性：イデアル格子上の最短ベクトル問題の矜持が、Ring-LWEの秘密sを見つける問題に帰着されることを量子アルゴリズムを用いて証明した。
　・決定問題の困難性：秘密sを見つけることが難しいなら、Ring-LWEの結果がランダムな値と区別できないことを、古典的な手法で証明した。

　<イデアル格子>
　　一般的な格子は基底ベクトルに何の制約もなく完全にランダムな配置が可能であるが、イデアル格子は代数的な構造を持ち強い制約を持つ。
　　一つのベクトルが決まると、それを回転(巡回)させたベクトルも格子の中に存在する。
　　一般的な格子と比べて「構造」を持つため、最悪ケースの最短ベクトル問題などで比較すると、イデアル格子の方がわずかに効率よく解けるとされている。
　　しかし、イデアル格子を使うと鍵サイズや計算速度で効率性が向上する他、「少し易しい可能性がある」とはいえ、最も効率的な攻撃手法を持ってしても、暗号を破るには非現実的な時間がかかる。
　　→ Ring-LWEではイデアル格子を採用

　<多項式をベクトルに変換する方法(カノニカル埋め込み)>
　　1.複素数を使う：方程式 x^n + 1 = 0 のn個の複素根 ζ1,ζ2,...,ζnとする。
　　2.写像(埋め込み)：多項式a(x)に対して、各根を代入した値の並びをベクトルとする。
　　　σ(a) = (a(ζ1),a(ζ2),...,a(ζn)) ∈ C^n
　　→ 多項式の掛け算 a(x)・b(x) をこの空間で見ると、ベクトルの成分ごとの掛け算（a(ζi)・b(ζi)）になる。
　　格子：多項式環Rの要素をカノニカル埋め込みσで飛ばした先の点集合が多次元空間R^nにおける格子点。共役成分を実部・虚部に分ける線形変換を施すことで、n次元の実数空間 R^n における点が得られる
　　エラー：誤差eは格子点から「わずかにズレた」ベクトルを表す。多項式を直接選ぶのではなく、先にカノニカル埋め込みされた空間で離散ガウス分布を考え、連続空間上の点を多項式の世界へ戻し、「係数が整数の多項式」としてノイズeを得る。

　<NTRU格子> 提案した論文：「Efficient Identity-Based Encryption over NTRU Lattices」
　　Ring-LWEによって生成される格子よりも強い数学的な構造を持つ。
　　Ring-LWEが「ノイズ(エラー)を加えた関係式」を安全性の根拠にするのに対し、NTRU格子は「多項式から作られる行列の短い基底を見つけるのが難しい」という幾何学的な困難性を安全性の根拠にしている。
　　
　　2つの多項式f,g(係数が小さく、秘密鍵になる)を定義する。
　　多項式環 Rq = Zq[x]/<x^n+1> において、公開鍵 h = g・f^{-1} (mod q)を計算する。
　　そして、NTRU格子L_{h,q}を、以下の条件を満たす多項式のペア(u,v)の集合と定義する。
　　　u・h ≡ v (mod q)
　　これをベクトル形式の行列(基底)で表すと、巡回行列(のブロック)の形になる。(1つの多項式hで表現できる)
　　R^n x R^n という2つの多項式空間を組み合わせた 2n次元の空間

　　※ NTRU格子では、秘密の多項式f,gを並べた行列が、そのまま格子の「非常に短くて直交に近い基底」になる。(2n次元の空間であるため、その中の点やベクトルを表すには必ず2つの多項式(要素)のペアが必要。)
　　→ Ring-LWEよりもさらに効率的に(高速で)「格子の点」をサンプリングできる他、公開鍵や暗号文のサイズを抑えることに成功。

　　~IDベース暗号への応用~
　　・ユーザーのIDをハッシュ関数を用いて多項式環Rqの要素(点c)にマッピングする。
　　・PKG(管理者)は、マスター秘密鍵(トラップドアである多項式f,g)を用いて、このターゲット点cの「すぐ近く」にあるNTRU格子の点vを探す。
　　　・サンプリング：トラップドアを用いてcに近い格子点v=(v1,v2)を離散ガウス分布に従ってサンプリングする。
　　　・格子点の条件：vは格子点であるので、NTRUの公開鍵hに対してv1・h ≡ v2 (mod q)を満たす。
　　・ユーザーの秘密鍵は、「ターゲット点cと、見つかった格子点vの距離(差分)」として定義される。
　　　　s1 = h_id - v1 (mod q)
　　　　s2 =  0   - v2 (mod q)
　　・(s1,s2)という多項式のペアが、ユーザーの秘密鍵になる。(s1,s2の係数は小さい) s1・h + s2 ≡ h_id (mod q)が成り立つ。

　　　*離散ガウス分布を効率的に行うための2つの技術
　　　1.(浮動小数点)高速フーリエ変換：多項式をカノニカル埋め込みの空間へ飛ばして、掛け算を「成分ごとの積」になる。計算量O(nlogn)
　　　2.ファルコン木(Falcon Tree)：NTRU格子の再帰的な構造を利用して、離散ガウスサンプリングで必要となる基底行列のGram-Schmidt直交化を2分木の形で考える。
　　　　　　　　　　　　　　　　　　　n次元の問題をn/2次元の2つの問題に分割し、それを葉の部分(1次元)まで繰り返して考える。
　　　→ メモリ節約、各階層で高速フーリエ変換することで計算の効率化 に成功

　<Falcon> 提案した論文：「Falcon: Fast-Fourier Lattice-based Compact Signatures over NTRU」
　　　NTRU格子を基盤にし、浮動小数点演算を伴うメッセージのサンプリングを安全に、効率的に実装するデジタル署名方法を提案
　　　→ 標準化された署名方式の一つ
